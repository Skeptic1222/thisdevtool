What Is URL Encoding?

URL encoding, also called percent-encoding, is a mechanism for representing special characters in a URL by replacing them with a percent sign (%) followed by two hexadecimal digits representing the character's ASCII code. For example, a space character (ASCII 32, hex 20) becomes %20, and an at sign (ASCII 64, hex 40) becomes %40. URL encoding is necessary because URLs are restricted to a subset of printable ASCII characters defined in RFC 3986. Characters outside this allowed set — including spaces, non-ASCII Unicode characters, and certain punctuation — must be encoded before being placed in a URL.

encodeURI vs. encodeURIComponent

JavaScript provides two functions for URL encoding, and choosing the right one matters:

  • encodeURI — encodes a complete URL. It leaves structural characters like /, ?, =, &, #, and : intact because they have structural meaning in a URL. Use this when you have a complete URL that you want to encode without breaking its structure.
  • encodeURIComponent — encodes a value that will be placed inside a URL, such as a query parameter value. It encodes everything that encodeURI encodes, plus the structural characters (/, ?, =, &, #, :, @). Use this when encoding individual query parameter names and values.

When to Encode URLs

You need to URL-encode strings in several common situations:

  • Query parameters — any value appended to a URL after the ? must be encoded if it contains spaces, ampersands, or other special characters
  • Form submissions — form data sent via GET requests is URL-encoded automatically by the browser, but you may need to encode values manually in JavaScript
  • API requests — when constructing API endpoints programmatically, always encode user-supplied values before embedding them in URLs
  • Redirect URLs — when including a full URL as a query parameter (e.g., a return URL after login), encode the entire URL using encodeURIComponent
  • Non-ASCII characters — URLs containing accented characters, emoji, or CJK characters must be encoded for reliable transmission across all browsers and servers

Common Percent-Encoded Characters

Here are some of the most frequently encountered percent-encoded characters:

  • Space → %20 (or + in form data)
  • Exclamation mark ! → %21
  • Hash # → %23
  • Dollar sign $ → %24
  • Ampersand & → %26
  • Plus sign + → %2B
  • Comma , → %2C
  • Slash / → %2F
  • Colon : → %3A
  • Equals sign = → %3D
  • At sign @ → %40

Double Encoding

A common mistake is double-encoding — running encodeURIComponent on a string that is already encoded. This turns %20 into %2520 (encoding the percent sign itself), which then causes errors when the server tries to decode the URL. Always check whether your string is already encoded before encoding it again. This tool's decode modes help you inspect what a URL actually contains before re-encoding it.

URL Encoding and Security

Proper URL encoding is also important for security. Failing to encode user-supplied values before embedding them in URLs can open your application to injection attacks. For example, if a user supplies a value containing & and you do not encode it, the value may be interpreted as a new query parameter. Similarly, unencoded values in redirect URLs can be exploited for open redirect attacks. Always validate and encode user input before using it in URLs.
